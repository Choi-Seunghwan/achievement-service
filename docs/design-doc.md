## Problem Statement

목표 관리 앱 시장에서 개인 유틸(루티너리)과 공동 목표(챌린저스)는 분리되어 있다.
개인 미션 관리 + 게이미피케이션(업적 시스템) + 커뮤니티(공개 업적)를 결합한 앱은 없다.

핵심 문제: 계획은 세우지만 실패를 반복하는 사람에게, **"나는 해내는 사람이다"라는 정체성을 확인**시켜주는 도구가 없다.

## Demand Evidence

- 창업자 본인이 타겟 유저: 포스트잇+노션으로 목표 관리 중이나 계획이 많아 잘 안 지켜짐
- 챌린저스(공동 목표 앱)와 루티너리(개인 루틴 앱) 모두 사용 경험에서 부족한 점을 구체적으로 식별
- 28세 여성 마케터 페르소나: 공개 업적 발견 → 참여 → 달성 → SNS 공유 → 성취감 루프 확인
- "항상 계획을 세우고 실패만 했으나, 달성한 걸 눈으로 보니 나도 잘하는 사람이었다" — 핵심 가치 문장

## Status Quo

| 솔루션 | 개인 유틸 | 게이미피케이션 | 커뮤니티 | 한계 |
|--------|----------|--------------|---------|------|
| 포스트잇/노션 | 강함 | 없음 | 없음 | 시각화/동기부여 없음 |
| 챌린저스 | 약함 | 약함 | 강함 (인증 기반) | 개인 할일 관리 불가 |
| 루티너리 | 강함 | 약함 | 약함 | 업적/공동 목표 없음 |

사용자들은 현재 포스트잇, 노션, 또는 위 앱들을 조합해서 쓰고 있으나, "달성한 것을 시각적으로 확인하고 공유하는" 경험은 없음.

## Target User & Narrowest Wedge

**타겟 유저:** 28세 여성, 커리어를 키워가는 IT 마케터. 트렌드에 민감. 러닝, 독서 등 자기개발에 관심. 인스타그램 활발 사용.

**핵심 시나리오 (Narrowest Wedge):**
1. 앱을 열면 "서울 마라톤 대회 준비" 같은 매력적인 공개 업적이 보임
2. 참여 버튼 누름 → 관련 미션들(주3회 러닝, 스트레칭 등)이 자동 생성됨
3. 매일 미션 완료 체크 → 업적 진행률 시각화
4. 업적 달성 → 인스타그램 스토리에 공유
5. "나는 해내는 사람이다"라는 성취감

**Wedge:** 매력적인 공개 업적 발견 → 참여 → 달성 → 공유

## Constraints

- 1인 개발 (풀스택)
- React Native (Expo) + NestJS + PostgreSQL 스택 고정
- 백엔드 모놀리식 통합 완료 (2026-03-23)
- Tamagui → NativeWind 마이그레이션 완료 (2026-03-23)
- 테스트 코드 없음, 디자인 시스템 미정립
- Pre-product 단계 (사용자 없음)

## Premises

1. **핵심 가치 = 성취감 제공 + 공유** — 미션 관리는 수단이고, 업적 달성+공유가 목적
2. **성장 엔진 = 공개 업적 콘텐츠 플랫폼** — 사용자가 업적을 발견하고 참여하는 게 핵심 루프
3. **초기 승부 = 콘텐츠 시딩 > 기술적 완성도** — 매력적인 공개 업적이 없으면 이탈

## Approaches Considered

### Approach A: 콘텐츠 퍼스트 — 공개 업적 중심 리빌드 ✅ 선택
- 공개 업적 발견/참여/달성/공유 루프를 핵심으로 정비
- 초기 콘텐츠 시딩 시스템 (30개+ 매력적인 업적)
- SNS 공유 기능 우선 개발
- 개인 미션 관리는 최소한으로 유지
- Effort: L | Risk: Med

### Approach B: 미니멀 바이어블 — 현재 코드 기반 정비
- 기존 코드 그대로, UX/디자인 폴리싱에 집중
- 테스트 코드 추가, 보안 강화
- 10개 정도 시드 업적 수동 생성
- Effort: S | Risk: Low

### Approach C: 소셜 콘텐츠 플랫폼
- 업적 생성 UX 극대화 (템플릿, 카테고리, 트렌딩)
- 사용자가 직접 매력적인 업적을 만들어 콘텐츠 공급
- 인스타그램 스토리 공유 기능 + 리더보드
- Effort: XL | Risk: High (치킨앤에그 문제)

## Recommended Approach

**Approach A: 콘텐츠 퍼스트**

핵심 전제(콘텐츠 시딩 > 기술적 완성도)에 가장 부합. 초기에 운영자가 직접 매력적인 업적을 시딩하고, 사용자는 "발견 → 참여 → 달성 → 공유" 루프에 집중.

**우선순위 (Must/Should 구분):**

**Must — 출시 필수:**
1. **카테고리 시스템 DB 마이그레이션** — PublicAchievement에 category 필드 추가 (S, ~2일)
2. **공개 업적 발견 UX 개선** — MVP: 카테고리별 탐색만 구현. 트렌딩 섹션과 참여자 수 표시는 Should로 이동 (M, ~1주)
3. **콘텐츠 시딩 Phase 1** — 10개 핵심 업적 + Prisma seed 스크립트 (M, ~1주 — 기획 포함)

**Should — 출시 후 2주 내:**
4. **업적 달성 시각화** — 진행률 바 + 달성 축하 화면 + 공유용 달성 카드 UI (M, ~1주)
5. **탐색기 추가 개선** — 트렌딩 섹션, 참여자 수 표시 (S, ~3일)
6. **SNS 공유 기능** — 스파이크 POC 선행 → 달성 카드 이미지 렌더링 → 네이티브 공유 시트 (M, ~1주)

**Won't (이번 스코프 아님):**
- 개인 미션 관리 변경 — 현재 수준 유지

**Effort 기준 (1인 개발, CC+gstack 활용):**
- S = ~1주 | M = ~2주 | L = ~1달 | XL = ~2달+
- Must 항목: ~3주 | Should 항목: ~2주 추가
- 전체: L+ (약 5주, Must만이면 ~3주에 출시 가능)

## 콘텐츠 시딩 전략

**방식:** 운영자가 직접 기획 + Prisma seed 스크립트로 DB에 투입. 초기에는 수동 기획이 핵심.

**카테고리 시스템:** 도입한다. 공개 업적에 카테고리 필드를 추가하여 탐색을 구조화.
- 운동/건강 (러닝, 마라톤, 홈트레이닝, 물 마시기)
- 독서/학습 (작가별 독서, 분야별 독서, 자격증 준비)
- 자기개발 (사이드 프로젝트, 외국어, 저널링)
- 라이프스타일 (요리, 여행, 절약)
- 시즈널/트렌드 (서울 마라톤 2026, 새해 목표)

**단계별 시딩 계획:**
- Phase 1 (출시 전): 10개 핵심 업적 (카테고리당 2개, 업적당 미션 3-5개 = 약 30-50개 미션)
- Phase 2 (출시 후 2주): 20개로 확대 (사용자 반응 높은 카테고리 중심)
- Phase 3 (출시 후 1달): 30개+ (시즈널 업적 추가)

**업적 기획 가이드라인:**
- 업적 1개 = 미션 3-5개 (너무 적으면 성취감 부족, 너무 많으면 포기)
- 미션 완료 기간 = 1-4주 (길면 이탈)
- 구체적이고 측정 가능한 미션명 (X: "운동하기", O: "5km 러닝 완주")

## SNS 공유 기술 검증

**접근법:** 클라이언트사이드 이미지 생성 → 네이티브 공유 시트
- `react-native-view-shot` — React Native 뷰를 이미지로 캡처
- `expo-sharing` — 네이티브 공유 시트 호출 (인스타그램 스토리 포함)
- 달성 카드 = 앱 내에서 달성 정보를 표시하는 전용 View 컴포넌트를 이미지로 렌더링

**숨겨진 복잡도 인식:**
- 인스타그램 스토리 직접 공유는 iOS/Android 별 API 차이 있음
- 이미지 생성 품질/해상도 테스트 필요
- **스파이크 태스크로 분리:** 구현 전 2시간 기술 검증(POC) 선행

## What I noticed about how you think

- "항상 계획을 세우고 실패만 했으나, 달성한 걸 눈으로 보니 나도 잘하는 사람이었다" — 이 문장에서 앱의 핵심 가치를 가장 정확하게 표현했습니다. 기능 목록이 아니라 감정으로 가치를 설명할 수 있다는 건 좋은 신호입니다.
- 챌린저스와 루티너리의 한계를 "개인 유틸 vs 커뮤니티"라는 축으로 정확하게 분류했습니다. 경쟁 제품을 카테고리 수준이 아니라 구조적 한계로 이해하고 있습니다.
- 구체적 유저를 물었을 때 처음에는 카테고리로 답했지만, 푸시 후 28세 마케터의 구체적 시나리오를 떠올렸습니다. 이건 타겟이 머릿속에 있긴 한데, 아직 그 사람과 직접 대화한 적은 없다는 뜻일 수 있습니다.
